End-to-End Azure ETL Pipeline for E-Commerce Analytics

The problem

E-commerce data arrived from two very different places — a public dataset pulled via HTTP from GitHub, and an internal SQL table — with no shared structure, no enrichment, and no path to a report a stakeholder could trust.

The approach

Azure Data Factory orchestrates ingestion from both sources into ADLS Gen2 as raw data. Azure Databricks transforms that raw data, enriching it with a lookup table pulled from MongoDB, and writes the transformed output back to a second ADLS Gen2 layer. Azure Synapse sits on top, exposing the transformed data for reporting through Power BI, Tableau, and Microsoft Fabric.

What I'd do differently

The MongoDB enrichment step introduces a dependency between two very different storage paradigms — document and lakehouse — inside one transformation stage. Next time I'd isolate that lookup as its own versioned reference dataset in ADLS, so the Databricks job doesn't need a live connection to MongoDB at run time.

  • Storage ADLS Gen2 (raw & transformed)
  • Transform Azure Databricks + MongoDB enrichment
  • Serving Azure Synapse → Power BI · Tableau · Fabric

MSc Thesis — Forecasting Welfare Recipients & Simulating Poverty Impacts of Policy Uplifts in Ireland

The problem

Budget 2026 announced a €10-per-week increase to core social welfare payments in Ireland, effective January 2026. The intention was clear; the expected impact on poverty rates was not. This research set out to forecast welfare recipient numbers for 2026 and quantify what the uplift would actually do to national poverty levels.

The approach

Using Ireland's quarterly Live Register data (2014–2025) from the Irish Government open data portal, I engineered lag-based features and trained three forecasting models — SARIMA, Prophet, and XGBoost — on data through 2022Q4, testing on 2023Q1–2025Q3. XGBoost won clearly on every metric and became the forecasting engine for 2026. I then built a policy simulation layer: using a poverty-income elasticity of −7.14 (projected from official 2023–2024 statistics), I converted the €10 weekly uplift into a projected quarterly poverty-rate change and government cost, for each of the four 2026 quarters the model forecast.

Actual (2014–2025) XGBoost forecast (2026)
Quarterly welfare recipients declining from roughly 391,000 in 2014Q1 to around 171,000 by 2025Q3, with a temporary rise around 2020. XGBoost forecasts 2026 quarters at approximately 178,935, 174,256, 174,747, and 168,394 recipients.

What I'd do differently

XGBoost was trained on only two lag features, with no macroeconomic or demographic variables and no explicit seasonal component. The elasticity-based poverty simulation also assumes a uniform income effect across all recipients, which glosses over household structure, secondary income, and regional cost-of-living variation. Next time I'd bring in unemployment and inflation series as exogenous features, and model heterogeneity in the elasticity itself rather than applying one national figure to every recipient.
